Consumer lag is the gap between what Kafka has produced and what your consumers have
processed. Left unmonitored, growing lag means stale downstream data, memory pressure,
and silent failures when consumer groups die. **Klag continuously monitors every
consumer group and ships the metrics to your observability stack.**

:::tip[Scales to large clusters]
Monitors thousands of consumer groups in ~50 MB heap. Request batching with configurable
delays fetches offsets for 500+ groups without spiking broker CPU.
:::

## Why Klag?

<CardGrid>
  <Card title="Lag velocity" icon="random">
    Know if lag is **growing or shrinking**, and catch problems before they escalate.
  </Card>
  <Card title="Time-based lag" icon="seti:clock">
    See lag in **seconds/minutes**, with time-to-close estimates, instead of raw message counts.
  </Card>
  <Card title="Hot partition detection" icon="warning">
    Find partitions with **uneven load** causing bottlenecks, via statistical outliers.
  </Card>
  <Card title="Group state tracking" icon="approve-check">
    Alert on **Rebalancing, Dead, or Empty** consumer-group states.
  </Card>
  <Card title="Data loss prevention" icon="error">
    Alert **before lag exceeds retention** and messages are lost forever.
  </Card>
  <Card title="Request batching" icon="setting">
    Safely monitor **large clusters** without overwhelming brokers.
  </Card>
</CardGrid>

## Get started in one command

```bash
docker run -e KAFKA_BOOTSTRAP_SERVERS=kafka:9092 \
           -e METRICS_REPORTER=prometheus \
           -p 8888:8888 \
           themoah/klag:latest
```

Metrics land at `http://localhost:8888/metrics`. See the [Quick Start](/getting-started/quick-start/)
for native image, Helm, and Docker env-file options.

## Built for AI-native operations

Klag exposes an opt-in, read-only [MCP endpoint](/ai/mcp/) so SRE and dev agents can
query lag, find lagging groups, and run composite `diagnose` checks, served from an
in-memory snapshot, never touching your brokers. A portable [Agent Skill](/ai/agent-skill/)
ships in the repo for Claude Code, Codex, and other harnesses.
